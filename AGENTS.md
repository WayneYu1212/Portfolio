# AGENTS.md

## 1. 项目身份

这是余宇颖的个人主页与数字名片，不是“简历网页化”，也不是面向单一岗位的求职落地页。

网站需要让访问者感受到她是一个具有研究能力、内容能力、互联网感知力和个人审美的人。求职价值应通过作品、判断、方法和人格自然显现，不要直接罗列“求职意向”，也不要把首页写成岗位关键词集合。

核心形象：

> 一个从历史、人物、社群和技术之间寻找故事，并把复杂材料转化为内容与体验的年轻创作者。

网站可以体现她对 AI、互联网和游戏的长期兴趣，但不要写成“我想进入 AI、互联网、游戏行业”的宣言。

## 3. 设计定位

视觉气质由三部分融合：

### A. 作品集原有视觉基因

保留并网页化：

- 米白、浅灰或纸张感背景
- 黑色粗线、像素边框和编辑标记
- 青蓝色主强调色
- 荧光黄次强调色
- 像素字体与现代中文无衬线字体混排
- 鼠标、窗口、选择框、编号、标签等电脑界面语言
- 中英文混排
- 活泼、轻巧、有个人手作感

### B. GeekOnUp 式第一印象

借鉴其抓眼球的首屏、沉浸感、镜头感、层次和滚动节奏，但只借鉴交互原则，不复制布局、品牌素材、文案和视觉资产。

首屏应有“打开后会停留几秒”的效果，但不应变成技术炫技展。

### C. Anthropic 式内容体验

正文区域借鉴其舒展、安静、留白充足、阅读层级清楚的感觉。

进入项目内容后，页面要从“酷”逐渐过渡到“舒服、可信、容易阅读”。

最终原则：

> 首屏负责吸引，正文负责建立信任，细节负责留下记忆。

## 5. 首页叙事

### 首屏

首屏概念为“余宇颖的数字房间 / 创作桌面 / 个人宇宙”。

应包含：

- 姓名：余宇颖 / YU YUYING
- 一句具有个人性的介绍
- 进入作品的入口
- 发送邮件入口
- 可选的声音开关，但默认静音
- 若有简历下载，只作为次级入口

首屏可以出现具有景深的物件：

- 录音机、采访话筒或波形
- 档案、书页、纸张
- 游戏手柄或游戏界面碎片
- 耳机、磁带、唱片或广东歌相关元素
- 相机、视频时间轴
- 鼠标、AI对话框或代码窗口
- 地图、印章和田野调查痕迹

这些物件不是技能图标墙。它们应该像一个人的真实兴趣和经历在空间中留下的痕迹。

### 首屏交互

优先使用：

- CSS 3D Transform
- 分层 SVG / PNG
- 鼠标视差
- 柔和的镜头推进
- 元素遮挡与景深
- 滚动时空间由散乱逐步聚合

Three.js 仅在确有必要时使用。

禁止：

- 无意义粒子宇宙
- 大量旋转球体
- 赛博朋克霓虹
- 紫蓝色 AI 模板
- 长时间 loading
- 强制观看开场动画
- 滚轮劫持
- 鼠标跟随造成卡顿或眩晕

## 6. 内容章节

### 6.1 Selected Work

项目是网站的核心。建议精选 4–6 个，而不是把所有经历全部放上去。

优先项目：

1. 柯子岭客家村口述史
2. Chronicle / 佣书 AI 游戏或互动叙事项目
3. 小红书游戏内容创作
4. 南越王博物院观众研究
5. AI 辅助历史研究工作流
6. 长湴村文旅策划或其他最成熟项目

每个项目列表页只回答：

- 这是什么
- 她做了什么
- 最重要的判断是什么
- 结果是什么
- 为什么值得点开

项目详情建议结构：

1. Context
2. My Role
3. Question
4. Process
5. Key Insight
6. Output
7. Result
8. Reflection

避免把项目详情写成简历条目扩写。

### 6.2 About

About 不是个人简历。

可写：

- 历史学训练如何影响她观察人与材料
- 为什么喜欢互联网、游戏和 AI
- 她如何在研究、写作、视觉和技术之间工作
- 目前正在探索什么

文字应真实、具体、有个人口吻，避免宏大空话。

### 6.3 Favorites

这是电子名片的重要组成部分，不是无关内容。

可以设置：

- 最近在听
- 广东歌歌单
- 喜欢的乐队和歌曲
- 最近在玩
- 最近在读
- 一些持续影响她的作品

广东歌部分可以使用唱片、磁带、播放器或电台界面表达。

允许展示：

- Supper Moment
- Pandora
- KOLOR
- Dear Jane
- Mr.
- 用户最终确认的歌曲与歌单

不要未经确认嵌入受版权保护的音频文件。优先使用外部音乐平台链接、合法嵌入或纯视觉播放器。

Favorites 应保持轻量，不能抢走作品区的主次。

### 6.4 Contact

只保留：

- Send an email
- GitHub
- 其他用户确认公开的平台

联系按钮使用 `mailto:` 或清晰的邮件交互。

不在网页正文公开：

- 手机号
- 出生年月
- 年龄
- 详细住址
- MBTI

## 7. 文案原则

文案应像一个真实的人，而不是职业测评报告。

推荐语气：

- 清楚
- 克制
- 有观察
- 有一点幽默
- 不端着
- 不过度自我包装

避免高频使用：

- 赋能
- 全链路
- 闭环
- 抓手
- 复合型人才
- 热爱互联网
- 深耕
- 极致
- 颠覆
- 斜杠青年

首页标题不要直接写岗位名称。

可探索的首页句式：

- 我收集人的故事，也研究它们如何被看见。
- 在历史、游戏、互联网和 AI 之间做一些具体的东西。
- I research people, shape stories, and build small worlds.
- Stories, systems, and things I care about.

最终文案必须由用户确认。

## 8. 动效规则

动效分为三个等级。

### Level 1：默认必须实现

- 首屏轻量视差
- 元素进入动画
- 项目卡片 hover 层次
- 导航状态
- 页面过渡
- 图片揭示

### Level 2：素材允许时实现

- 桌面物件景深
- 滚动镜头推进
- 项目档案展开
- 音乐播放器视觉交互
- 鼠标靠近元素时产生局部响应

### Level 3：用户确认后再实现

- Three.js 场景
- WebGL 图片变形
- 复杂 shader
- 自定义物理交互
- 大型 3D 模型

不要在第一版直接实施 Level 3。

所有动画必须：

- 支持 `prefers-reduced-motion`
- 移动端自动简化
- 不阻塞阅读
- 不影响键盘访问
- 失败时仍能访问全部内容

## 9. 技术要求

推荐：

- React
- TypeScript
- Vite
- CSS Modules 或清晰的 CSS 变量体系
- GSAP 仅用于关键滚动动画
- Three.js / React Three Fiber 仅用于必要场景
- 静态数据文件管理内容

内容数据建议：

- `src/data/profile.ts`
- `src/data/projects.ts`
- `src/data/favorites.ts`
- `src/data/socials.ts`

项目数据不要散落在组件中。

必须满足：

- GitHub Pages 可部署
- 正确配置 Vite `base`
- GitHub Actions 自动部署
- 响应式
- 移动端可用
- 基础 SEO
- Open Graph
- 图片懒加载
- 语义化 HTML
- 键盘可访问
- 可见焦点状态
- alt 文本
- WebGL 降级
- reduced motion
- 不因动画库加载失败而丢失正文

## 10. 工作方式

每次接到任务时：

1. 先检查现有代码和资料。
2. 简述理解。
3. 列出本次计划修改的文件。
4. 先做最小可用版本。
5. 执行构建和测试。
6. 报告实际结果。
7. 不要顺手重构无关代码。
8. 不要一次加入多个高风险动画。
9. 不要删除用户已有素材。
10. 视觉方向不明确时，用静态草图或低成本原型先验证。

当用户提出视觉参考网站时：

- 先分析其布局、交互、节奏、技术和可迁移元素
- 只提取原则，不做像素级克隆
- 检查源码许可
- 不复制品牌素材、摄影、图标、字体和文案
- 明确说明哪些效果适合本项目，哪些不适合

## 11. 第一阶段交付

首次运行时不要直接完成整站。

先输出：

1. 对作品集视觉语言的总结
2. 网站核心概念
3. 首页线框结构
4. 内容章节
5. 首屏交互分镜
6. GeekOnUp 与 Anthropic 的融合方式
7. 项目筛选建议
8. 需要用户补充的素材
9. 技术路线
10. 开发阶段划分

用户确认后，再开始实现静态版本。

## 12. 验收标准

最终网站应让访问者在 10 秒内感受到：

- 她有明确的个人气质
- 她与 AI、互联网、游戏的关系是真实而自然的
- 她不只是“会写文案”
- 她能研究人、理解社群、整理复杂信息并做出作品
- 她有自己喜欢的音乐、游戏和文化
- 网站本身就是她审美与学习能力的证明

最终效果不是“最炫的网站”，而是：

> 酷到让人愿意停下，舒服到让人愿意看完，具体到让人相信她真的做过。
